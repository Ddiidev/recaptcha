# recaptcha

V module for validating Cloudflare Turnstile tokens with the Siteverify API.

Portuguese documentation: [README-ptbr.md](README-ptbr.md)

## Project scope

This project started as a small test to make Cloudflare Turnstile validation simple to integrate. It intentionally stays as a thin wrapper around Siteverify, even though it later became part of a real project.

## Installation

Add this module to a V project and import it as `recaptcha`.

```v
import recaptcha
```

## Usage

```v
import recaptcha

fn main() ! {
    response := recaptcha.new(
        token: 'client-response-token'
        secret_key: 'turnstile-secret-key'
    )!

    if !response.is_valid() {
        return error('invalid captcha')
    }

    if !response.is_action('login') {
        return error('unexpected captcha action')
    }
}
```

## API

### `ParamsNew`

Input for `new`.

| Field | Type | Description |
| --- | --- | --- |
| `token` | `string` | Token generated by the Turnstile widget on the client. |
| `secret_key` | `string` | Secret key used to validate the token with Cloudflare. |
| `expected_action` | `?string` | Optional field kept in the public contract. The current implementation does not send it to Cloudflare or validate it automatically. Use `RecaptchaResponse.is_action` when action validation is required. |

### `new(param ParamsNew) !RecaptchaResponse`

Sends `token` and `secret_key` to Cloudflare Siteverify and returns the decoded response.

The function returns an error when:

- the HTTP request fails;
- Cloudflare returns a non-200 status code;
- the response body cannot be decoded as `RecaptchaResponse`.

### `RecaptchaResponse`

Decoded response returned by Cloudflare.

| Field | Type | Description |
| --- | --- | --- |
| `success` | `bool` | Whether Cloudflare accepted the token. |
| `challenge_ts` | `string` | Timestamp returned by Cloudflare for the challenge. |
| `hostname` | `string` | Hostname associated with the validation. |
| `error_codes` | `[]string` | Error codes returned by Cloudflare. |
| `action` | `string` | Action returned by Cloudflare, when configured in the widget. |
| `cdata` | `string` | Custom data returned by Cloudflare, when provided. |

### `is_valid() bool`

Returns `true` when `RecaptchaResponse.success` is `true`.

### `is_action(action string) bool`

Returns `true` when `RecaptchaResponse.action` is exactly equal to `action`.

## Notes

- This module does not change the response returned by Cloudflare.
- Action validation is explicit. Call `is_action` after `new` if your application depends on a specific action.
- Keep the secret key on the server side. Do not expose it in client-side code.
